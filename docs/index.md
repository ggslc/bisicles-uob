# BISICLES Documentation

## User Documentation

This documentation aims to show how to build and run BISICLES, and to select run-time options. Most of it will make more sense in the context of the published papers. To begin with we suggest [Cornford, Martin et al, The Cryosphere, 9, 1579-1600, 2015](http://www.the-cryosphere.net/9/1579/2015/), which describes the steps needed to run realistic problems, tuned to modern satellite and airborne observations.

The driver, solverBenchmark, and cdriver interface share many run-time options:

- [Configuration, input, output, and checkpoints](configuration/io.md)
- [Ice sheet geometry](configuration/geometry.md) - specifying initial ice thickness, bedrock elevation, and boundary conditions
- [Stresses (in-ice and basal)](configuration/stress.md) - specifying stress model components
- [Thermodynamics](configuration/thermodynamics.md) - initial state and evolution of internal energy density
- [Calculation of ice velocity](configuration/velocity.md) - includes forward and inverse problems
- [Surface fluxes](configuration/surfaceflux.md) - accumulation, ablation, energy balance, bedrock uplift
- [LevelData interface](configuration/leveldatainterface.md) - load meshed data
- [Python interface](configuration/pythoninterface.md) - compute data on the fly with simple formulae

## Developer Documentation

Developer documentation is included with the source distribution. Developers should be familiar with C++ and [Chombo](https://commons.lbl.gov/display/chombo/Chombo+-+Software+for+Adaptive+Solutions+of+Partial+Differential+Equations).

If you have doxygen and graphviz installed, documentation can be generated by running:

```bash
cd path-to-bisicles/code
make doxygen
```

This will produce html files in `path-to-bisicles/code/doc/doxygen/html/index.html`.

Design documents (`.tex` files) are in `path-to-bisicles/code/doc/`. PDF files can be created by running:

```bash
cd path-to-bisicles/code
make doc
```

There are also some tests developers need to be aware of:

- [Unit tests](tests.md) - tests of individual components
- [Regression tests](regression.md) - tests of the code as a whole
